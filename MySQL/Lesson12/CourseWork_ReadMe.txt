Тема курсовой работы: база данных для системы управления тестами (не важно какими пока).

Объекты:
- Тестируемый объект = Unit (типы мобильных приложений) определенного типа (под который пишутся тесты)
  Типичный тест подходит к любому юниту определенного типа (тест совместимости SNS с OS) 
- Тест (текстовое описание или текст-скрипт), подходящий к определенному типу объектов. К тесту может быть прикреплен файл
- Файл = вспомогательный к тесту материал
- Тестовый проект = группа тестов. Она же - категория тестов.
- Пользователь системы с пока фиксированной ролью: su, admin, tester. Ну, и роли, соответственно
- Расписание = Начало-Конец какой-то процедуры. Конца может не быть - тогда тест идет до упора.
- Тестовая задача = маппинг теста на объект подходящего типа и расписание выполнения задачи. Можно указать кол-во повторов задачи
- Результат = результат исполнения итерации тестовой задачи
- Нотификации = системные нотификации конкретному юзеру

Использованы
- Триггер в таблице tasks_results, который оставляет сообщение об остановке тест задачи тестеру, который начал тест задачу
- Функция launch_test_task, которая "запускает" тест, помещая запись об исполнении тест задачи в tasks_results
  SELECT launch_test_task(10, 3); запускает итерацию 3 тест-задачи 10
